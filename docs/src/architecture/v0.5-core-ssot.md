# Vox v0.5 Core Architecture SSOT

## Status: Preliminary for v0.5 (Experimental)
**Date**: 2026-04-18
**Authority**: Vox Release Engineering

This document defines the authoritative architecture for the Vox v0.5 release. **Note: Vox is currently in a heavy state of continuing development and is NOT yet ready for a 1.0 release.** Research and exploratory findings from the development phase (Q1 2026) have been moved to the archive (`docs/src/archive/research-2026-q1/`).

## 1. Core System Components

### 1.1 Vox CLI (`vox-cli`)
- **Role**: Primary command surface and entry point for all operations.
- **Dispatch**: Unified Latin-named command hierarchy.
- **CI Guards**: Strictest enforcement of `ssot-drift`, `toestub`, and `command-compliance`.
- **Feature Matrix**: Default is `mens-base`. Optional `gpu` feature for native training.

### 1.2 Vox DB (`vox-db`)
- **Role**: Durable persistence layer using Turso/SQLite.
- **Key Schemas**: Managed via SQLx migrations.
- **Data-Plane Guards**: `query-all-guard` ensures query integrity across the workspace.

### 1.3 Vox Mens (`vox-mens` / `vox-populi`)
- **Role**: Intelligence layer.
- **Training**: Burn-powered QLoRA (Candle-based dispatch).
- **Inference**: Native Rust inference with Axum-based serving.
- **Protocol**: Aligned with the Mens synthesis research.

### 1.4 Vox DEI / Orchestrator
- **Role**: Agentic orchestration and distributed execution.
- **Protocol**: MCP-native tools and Symphony-mode visualizer.

## 2. Structural Invariants

- **Single Source of Truth (SSOT)**: All architectural claims must be grounded in `canonical-map.v1.yaml` and verified by `vox ci ssot-drift`.
- **TOESTUB Governance**: Minimum structural limits (line counts, module depth) are strictly enforced in `docs/agents/governance.md`.
- **Non-Null Boundary**: No `null` values allowed across machine-readable interfaces (YAML, JSON) unless explicitly documented in a versioned schema.

## 3. Deployment Pipeline ("Golden Path")

- **Build**: `vox build` produces unified OCI-compliant artifacts or bare-metal binaries.
- **Deploy**: `vox deploy` with multi-target support (Fly.io, Bare Metal, Compose).
- **Validation**: Continuous validation via `marquee_app` case study.

## 4. Documentation Strategy

- **Active Docs**: Reference guides in `docs/src/reference/` and `docs/src/how-to/`.
- **Archived Research**: 270+ research documents tombstoned in `docs/src/archive/research-2026-q1/` to prevent LLM context pollution and maintain architectural focus.
