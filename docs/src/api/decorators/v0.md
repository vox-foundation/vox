# @v0

**Category:** ui

## Syntax

```vox
@v0 "Screen description for the model" fn Dashboard() to Element

@v0 from "design/mockup.png" fn Landing() to Element
```

## Description

Declares a **React** surface whose implementation is produced by **[v0.dev](https://v0.dev)** when **`V0_API_KEY`** is available (via [Clavis](../../reference/clavis-ssot.md)). During **`vox build`**, if **`{Name}.tsx`** is missing under the TypeScript output directory, the CLI calls the v0 API and writes the file; if the file already exists, it is left unchanged.

## Named exports (required for `routes:`)

TanStack **`routes:`** codegen imports screens as **named** bindings: `import { Dashboard } from "./Dashboard.tsx"`. v0 sometimes returns **`export default`**; the CLI normalizes to **`export function Name`** in [`v0_tsx_normalize.rs`](../../../crates/vox-cli/src/v0_tsx_normalize.rs).

**`vox build`** fails if an `@v0` component’s **`{Name}.tsx`** is on disk but breaks this contract (for example only a default export). Regenerate or hand-fix so one of these holds: **`export function Name`**, **`export async function Name`**, **`export const Name`**, or **`export { Name }`**.

## Optional: `vox doctor`

Set **`VOX_WEB_TS_OUT`** to the same directory **`vox build` uses for `*.tsx`**. [`vox doctor`](../../reference/cli.md) then checks each **`@v0`** declaration found under the current tree against the corresponding file. See [`env-vars.md`](../../reference/env-vars.md#web--vite--tanstack-codegen).

## Related: `@island` and `vox island`

For **client-only React islands** (separate Vite bundle, hydration), use **`@island`** and the **`island`** CLI feature—not **`@v0`** on its own. v0-generated TSX can still feed **`vox island`** workflows when you want AI-generated markup inside an island package. Entry points: [vox-web-stack SSOT](../../reference/vox-web-stack.md), [`cli.md`](../../reference/cli.md) (`vox island`).
