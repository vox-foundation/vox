# `@v0`

The `@v0` decorator bridges the gap between AI-generated UI prototypes and the Vox compiler. It instructs the compiler to retrieve a completed generative interface via Vercel's `v0` CLI and lower it directly into a standard Vox `@island`.

## Syntax

```vox
@v0 "chat-id" 
fn ComponentName() -> Element
```

- **"chat-id"**: The deterministic ID assigned to your component on v0.dev (e.g., `"yM1xXq6"`).
- **ComponentName**: The name the component will be imported under in Vox apps. This must match the exported name.

## Build-Time Codegen Effect

When you run `vox build`, the compiler performs two sequential steps:

1. **Sidecar Generation**: Vox spawns a background `npx v0 add <chat-id> --name ComponentName` command. The Vercel v0 CLI connects to Vercel, resolves dependencies (like Shadcn UI parts), and safely writes the component to the specified output directory.
2. **Web/AST Lowering**: Vox seamlessly treats the newly-acquired file exactly as it would treat an `@island`. The compiler extracts the interface bindings and bridges it into the Rust-compatible executable target.

> [!TIP]
> Because Vox delegates to the authenticated `v0` CLI to retrieve the file rather than hitting an unauthorized HTTP endpoint, you must be logged into the v0 CLI using `npx v0 login` in your terminal prior to running `vox build`.

## Limitations & Future Work

Currently, the `v0` CLI only supports downloading pre-generated components via their `chat-id` strings. It does not support native prompting or image uploads directly from the command line.

**To-Do**: In the future, we plan to extend Vox to support multimodal prompt injection directly within the `@v0` tag (e.g., `@v0 prompt="Match this layout" image="./design.png"`). Because the official CLI lacks this endpoint, this will likely require piping image-to-text through a multimodal model like Gemini or Nano to programmatically synthesize Harmonious UI directives.
